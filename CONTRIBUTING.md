# Contributing to Deck & Dice Studios

Thank you for considering contributing to **Deck & Dice Studios**! Whether it's a bug fix, feature request, or documentation improvement, we appreciate your interest and help in making the project better.

## How You Can Contribute

### 1. **Reporting Bugs**  
   - If you find a bug, please check the [Issues](https://github.com/yourusername/yourrepo/issues) to see if it's already reported.
   - If not, create a new issue with a description of the bug and steps to reproduce it.

### 2. **Suggesting Features**  
   - Feature suggestions are welcome! Please create a new issue and label it as a **Feature Request**.
   - Try to describe your idea clearly with examples of how it would benefit the project.

### 3. **Submitting Code**  
   - Fork the repository and create a new branch (`git checkout -b feature-branch`).
   - Make your changes and ensure tests pass before submitting a pull request.
   - Write clear and concise commit messages.
   - Ensure that your code adheres to the project's style guidelines.
   - If you're adding a new feature, make sure it's well-documented.

### 4. **Writing Documentation**  
   - Documentation improvements are always welcome! This includes updates to README, API docs, or code comments.
   - Please review your changes for clarity, spelling, and grammar.

### 5. **Reviewing Pull Requests**  
   - If you prefer not to submit code but want to help review others' contributions, feel free to review open pull requests.
   - Please provide constructive feedback.

---

## Code of Conduct

By participating in this project, you agree to uphold the standards of respectful communication and contribute positively. Be kind, respectful, and considerate when interacting with others.
